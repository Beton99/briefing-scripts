# 盘后报告结构契约

最终文件名：`YYYY-MM-DD美股盘后报告.md`。行情表由 `yf.py` stdout 直接输出（含各组 markdown 表格），AI 只写基于搜索的定性叙述。表格用 Markdown，列照抄底稿表头，不增删不改数。

## yf.py 字段映射

| 脚本字段 | 报告列名 |
|---------|---------|
| 延时价 | 盘后价 |
| 延时（涨跌幅） | 盘后涨跌幅 |
| 收盘 | 今日收盘 |
| 涨跌幅 | 涨跌幅（默认当日） |
| 前收 | 前一日收盘 |

非美股价格表以 yf.py 返回的市场日期/最新交易日为准。

## 表格格式

- 代码列格式：`` `代码`<br>`*简称*` ``
- 章节标题只写名称
- 数据表用 Markdown 表格，数值照抄脚本不改

## 固定结构（14节）

1. **H1 标题**：`YYYY-MM-DD美股盘后报告`
2. **抬头三行**（H1 下紧接）：
   ```
   > 美股交易日：<美股交易日>（周X，美东夏/冬令时）｜已收盘
   > 数据源：Yahoo Finance 批量(repo)｜取数状态：成功N只
   > 报告生成：<北京时间> 北京时间
   ```
3. `复盘重点` — 强制模板：
   - **今日主线：** 粗体长句，把当日最核心市场矛盾讲清楚
   - **核心驱动要点：** 有几点写几点（①/②/③...），每点 `**① 标题。**` 整句粗体+展开
   - **跨市场传导：** 独立段落，粗体标签开头
   - **盘后关键变量：** 独立段落，粗体标签开头
   - **次日验证锚：** 独立段落，粗体标签开头，用"→若X则Y"条件式
4. `一、收盘全景` — 两个表格：
   - 美股指数表（5行）：代码|名称|收盘|涨跌幅
   - 全球指数表（9行）：代码|名称|收盘|涨跌幅
   - 表格下紧接 CNN 恐贪行 + 三段文字（市场宽度与风格、波动与利率传导、财报季）
5. `二、板块归因` — 领涨/领跌板块+原因+风格切换信号
6. `三、S&P500行业` — 表列序：代码|行业|收盘|涨跌幅|前收|前收涨跌|1个月|3个月|1年|2年；表后用小结写领涨/领跌/资金风格
7. `四、AI产业链` — 14环节表列序：环节|代表|名称|收盘|涨跌幅|延时|延时涨跌|前收|前收涨跌|1个月|3个月|1年|2年
   - 环节名只写首行后续留空；代表栏只写代码；名称栏写中文简称
8. `五、大宗、汇率与加密` — 列序：品种|代码|实时|涨跌幅|前收|前收涨跌；分三段叙述
9. `六、大事记` — 回扣昨日关注项；明日数据/事件/财报+遗留风险
10. `★七、自选股行情` — 列序：代码|名称|收盘|涨跌幅|延时|延时涨跌|前收|前收涨跌|1个月|3个月|1年|2年。全部标的保留含非美股
11. `八、盘后异动与财报` — 盘后异动表列代码|名称|收盘|延时|延时涨跌幅|原因，仅延时涨跌幅超 ±2% 入选
    - **今日财报**用列表格式覆盖当天盘前+盘后所有重大财报
    - **明日重要财报**用列表格式列下一交易日盘前/盘后重要财报
12. `★九、关注股重大消息` — 逐字照搬深扫卡全文，每卡三段平级（`**最近发生了什么：**` / `**意味着什么：**` / `**一句话：**`）
    - 严禁缩写/浓缩/改写；严禁输出"未出卡标的"表格
    - 所有来源必须 `[媒体名](URL)` 格式
13. ETF板块脉搏 — 四组项目符号
14. 免责声明

## 铁律

1. 行情唯一来源=yf.py 脚本底稿，搜索只用于定性归因
2. 涨跌幅/价格照抄脚本，不重算、不改小数位
3. 新闻中的盘前/盘后涨跌幅不得当作价格依据
4. 第九节深扫卡禁止改写——逐字照搬三段+全部链接
5. 取数成功率<90%中止
6. 禁止交易语言（买入机会/逢低/抄底/减仓等）
7. 取不到写"未获取"，禁编造/填旧值/估算
8. 每句因果挂可点链接；无链接写"基于 yf.py 表格观察"
9. 来源规则严格执行：优先来源优先、二线标[二线来源]、低质标[来源质量警告]
10. 非美简称照锁定表，禁自行识别
11. 正常收盘正文顶部不写交易状态说明
12. 第八节盘后异动表仅用 yf.py 延时字段筛选超 ±2%
13. 报告中不含 TOKEN 计数/用量信息
14. 报告末尾不重复"报告生成时间/数据源/取数成功率"
