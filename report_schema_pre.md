# 盘前报告结构契约

最终文件名：`YYYY-MM-DD美股盘前报告.md`。底稿由 yf.py 输出（中性标签："延时价"/"延时"/"收盘"/"涨跌幅"/"前收"/"前收涨跌"），AI 在装配时按本文件的映射规则将底稿列映射到最终报告列名。表格用 Markdown，数值照抄底稿不改。

**必须逐列提取，不准自行增删列。必须丢弃底稿中不在映射中的列。**

## yf.py 字段映射

| 脚本字段 | 报告列名 |
|---------|---------|
| 延时价 | 实时价 |
| 延时（涨跌幅） | 实时价涨跌幅 |
| 收盘 | 昨日收盘 |
| 涨跌幅 | 昨日涨跌幅 |

非美股无盘前盘后交易："实时价/实时价涨跌幅"取脚本"收盘/涨跌幅"，"昨日收盘"取"前收"。

## 底稿代码列拆分

底稿代码列格式为 `` `代码`<br>`*简称*` ``（代码和简称合并在同一单元格）。AI 装配所有表格时，代码列只输出代码（如 `ES=F`），名称列输出简称。不得复制底稿的合并格式到报告。代码列中出现 `<br>*简称*` 即为不合格。

## 表格格式

- 代码列格式：`` `代码`<br>`*简称*` ``
- 章节标题只写名称
- 数据表用 Markdown 表格，数值照抄底稿不改

## 固定结构（13节）

1. **H1 标题**：`YYYY-MM-DD美股盘前报告`
2. **抬头三行**（H1 下紧接）：
   ```
   > 美股日期：<美股日期>（周X，美东夏/冬令时）｜盘前快照
   > 数据源：Yahoo Finance 批量(repo)｜取数状态：成功N只
   > 报告生成：<北京时间> 北京时间
   ```
   休市/半日市时抬头注明"半日市，提前收盘"或休市状态。
3. `★盘前重点` — 按以下模板输出：
   - **今日主线：** 粗体长句，不限一句，把盘前最核心市场矛盾讲清楚。需覆盖：隔夜复盘、盘前期指方向、板块分化预判。
   - **核心驱动要点：** 有几点写几点（①/②/③...），每点 `**① 标题。**` 整句粗体+展开。每点至少挂一个可点链接。
   - **验证锚：** 独立段落，粗体标签开头，用"→若X则Y"条件式
4. `一、隔夜与盘前` —
   - 指数期货表列序：代码|名称|实时价|实时涨跌幅|前收|前收涨跌
   - 三大指数+SOX+VIX 表列序：代码|名称|昨日收盘|昨日涨跌幅（丢弃 1M/3M/1Y/2Y 和前收列）
   - 表格下紧接 CNN 恐贪行（"未获取(需手动核实)"+链接），CNN 之后才写文字说明
   - S&P500 全 11 行业表，列序按上方映射。表后给小结写领涨/领跌/风格和持续性
5. `二、全球指数` — 列序按上方映射
6. `三、AI产业链` — 14环节表，列序按上方映射。表后按环节挑异动最明显 1-2 个简述。搜不到事件标记"无重大事件"
7. `四、宏观事件` — 短句标北京时间；CME FedWatch 写"未获取(需手动核实)"
8. `五、大宗、汇率与加密` — 列序按上方映射；分大宗/汇率/加密三段，每段给超链接。代码列强制改写（USD/CNH 等）
9. `六、大事记` — 今日财报用表格（代码|名称|实际EPS|预期EPS|公布时间|结果）；含未来7日重要财报列表。搜索范围=自选股+重要公司
10. `★七、自选股行情` — 列序按上方映射。全部标的保留含非美股不得删行。美股实时价涨跌超±2%的必须填异动原因（逐只搜索归因），未超留空。简称按锁定表。
11. `★八、关注股重大消息` — 逐字照搬深扫卡全文，每卡三段平级（`**最近发生了什么：**` / `**意味着什么：**` / `**一句话：**`）。严禁缩写改写；严禁输出"未出卡标的"表格。所有来源必须 `[媒体名](URL)` 格式。关注股中前一日盘后+当日盘前盘后已发布/即将发布财报的标的必须出深扫卡。深扫卡禁止输出内部标记 `[high]`/`[medium]`/`[low]`。
12. ETF板块脉搏 — 四组项目符号（动量/宽基、AI/半导体、能源/电力、金融/小盘）
13. 免责声明

## 铁律

1. 行情唯一来源=yf.py 脚本底稿，搜索只用于定性归因
2. 涨跌幅/价格照抄底稿，不重算、不改小数位
3. 新闻中的盘前/盘后涨跌幅不得当作价格依据
4. 第八节深扫卡禁止改写——逐字照搬三段+全部链接
5. 取数成功率<90%中止
6. 禁止交易语言（买入机会/逢低/抄底/减仓等）
7. 取不到写"未获取"，禁编造/填旧值/估算
8. 每句因果挂可点链接；无链接写"基于 yf.py 表格观察"
9. 来源规则严格执行：优先来源优先、二线标[二线来源]、低质标[来源质量警告]
10. 非美简称照锁定表，禁自行识别
11. 正常开市正文顶部不写交易状态说明；休市/未确认顶部注明
12. 代码列不含 `<br>` 残留（已拆分为代码和名称两列）
13. 深扫卡首条直接写日期和价格数据，不写"状态/价格行："或"价格锚"前缀
14. 三大指数及 SOX 做"前收×(1+涨跌)≈今收"自检，自检不写进正文
15. 非美股若脚本交易日不同于 market_context 对应日期，正文必须标注
16. 报告末尾不重复"报告生成时间/数据源/取数成功率"，抬头已有
