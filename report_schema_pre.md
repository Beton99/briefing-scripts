# 盘前报告结构契约

最终文件名：`YYYY-MM-DD美股盘前报告.md`。行情表由 `yf.py` stdout 直接输出（含各组 markdown 表格），AI 只写基于搜索的定性叙述。表格用 Markdown，列照抄底稿表头，不增删不改数。

## yf.py 字段映射

| 脚本字段 | 报告列名 |
|---------|---------|
| 延时价 | 实时价 |
| 延时（涨跌幅） | 实时价涨跌幅 |
| 收盘 | 昨日收盘 |
| 涨跌幅 | 昨日涨跌幅 |

非美股无盘前盘后交易："实时价/实时价涨跌幅"取脚本"收盘/涨跌幅"，"昨日收盘"取"前收"。

## 表格格式

- 代码列格式：`` `代码`<br>`*简称*` ``
- 章节标题只写名称
- 数据表用 Markdown 表格，数值照抄脚本不改

## 固定结构（13节）

1. **H1 标题**：`YYYY-MM-DD美股盘前报告`
2. **抬头三行**（H1 下紧接）：
   ```
   > 美股日期：<美股日期>（周X，美东夏/冬令时）｜盘前快照
   > 数据源：Yahoo Finance 批量(repo)｜取数状态：成功N只
   > 报告生成：<北京时间> 北京时间
   ```
3. `★盘前重点` — 以下模板强制：
   - **今日主线：** 粗体长句，把盘前最核心市场矛盾讲清楚
   - **核心驱动要点：** 有几点写几点（①/②/③...），每点 `**① 标题。**` 整句粗体+展开
   - **验证锚：** 独立段落，粗体标签开头，用"→若X则Y"条件式
4. `一、隔夜与盘前` —
   - 指数期货表列序：代码|名称|实时价|实时涨跌幅|前收|前收涨跌
   - 三大指数+SOX+VIX 表列序：代码|名称|昨日收盘|昨日涨跌幅（不写实时价）
   - S&P500 全 11 行业表列序：代码|行业|收盘|涨跌幅|前收|前收涨跌|1个月|3个月|1年|2年
   - 表格下紧接 CNN 恐贪行：名称"CNN 恐惧与贪婪指数"，数值"未获取(需手动核实)"，链接 https://www.cnn.com/markets/fear-and-greed
5. `二、全球指数` — 列序：代码|名称|收盘|涨跌幅|前收|前收涨跌|1个月|3个月|1年|2年
6. `三、AI产业链` — 14环节表，列序：环节|代表|名称|实时价|实时涨跌幅|昨日收盘|昨日涨跌幅
   - 环节名只写首行后续留空；代表栏只写代码不写名称；名称栏写中文简称
   - 实时价：美股=盘前实时价；昨日收盘=前一交易日收盘价
7. `四、宏观事件` — 短句标北京时间；CME FedWatch 写："未获取(需手动核实) — [CME FedWatch Tool](https://www.cmegroup.com/markets/interest-rates/cme-fedwatch-tool.html)"
8. `五、大宗、汇率与加密` — 列序：品种|代码|实时|涨跌幅|前收|前收涨跌；分大宗/汇率/加密三段，每段给超链接
9. `六、大事记` — 今日财报用表格（代码|名称|实际EPS|预期EPS|公布时间|结果）；含未来7日重要财报列表
10. `★七、自选股行情` — 列序：代码|名称|实时价|实时涨跌幅|前收|前收涨跌幅|1个月|3个月|1年|2年|异动原因
    - 代码列只写代码不写名称；名称列写中文简称
    - 全部标的保留含非美股不得删行
11. `★八、关注股重大消息` — 逐字照搬深扫卡全文，每卡三段平级（`**最近发生了什么：**` / `**意味着什么：**` / `**一句话：**`）
    - 严禁缩写改写；严禁输出"未出卡标的"表格
    - 所有来源必须 `[媒体名](URL)` 格式
12. ETF板块脉搏 — 四组项目符号（动量/宽基、AI/半导体、能源/电力、金融/小盘）
13. 免责声明

## 铁律

1. 行情唯一来源=yf.py 脚本底稿，搜索只用于定性归因
2. 涨跌幅/价格照抄脚本，不重算、不改小数位
3. 新闻中的盘前/盘后涨跌幅不得当作价格依据
4. 第八节深扫卡禁止改写——逐字照搬三段+全部链接
5. 取数成功率<90%中止
6. 禁止交易语言（买入机会/逢低/抄底/减仓等）
7. 取不到写"未获取"，禁编造/填旧值/估算
8. 每句因果挂可点链接；无链接写"基于 yf.py 表格观察"
9. 来源规则严格执行：优先来源优先、二线标[二线来源]、低质标[来源质量警告]
10. 非美简称照锁定表，禁自行识别
11. 正常开市正文顶部不写交易状态说明
12. 报告末尾不重复"报告生成时间/数据源/取数成功率"，抬头已有
